Chingu Hackathon

For the Chingu Holiday Hackathon 2021, teams were set the task of designing an app for Santa to use on Christmas Eve.

Eating too many cookies slows him down, so he has requested an app that will: let him assign a calorie target; track the calories consumed and burnt during the night; and monitor his progress on a dashboard that updates once per second.

Together with two friends, we built a responsive dashboard and, after finishing the minimum requirements, added some stretch goals of our own.

We collaborated with Git/GitHub and used Notion to manage the project.

Project Requirements

Minimum Hackathon Requirements

  • Teams may use any language, framework, or libraries
  • Create a UI/UX that's distinctive, fun, and appealing
  • Responsive on mobile and any screen size or orientation, and meet accessibility standards

The app must have a dashboard that allows Santa to enter a calorie target and then update a display every second with the following information:

  • calories accumulated so far
  • homes that give milk & cookies
  • homes that give carrots & tea
  • current delivery speed (homes per second)
  • homes visited
  • homes remaining
  • time remaining

Santa requests a start and stop button.

Stretch Goals

The creators of the Hackathon decided that if Santa hit 5,000 calories, he would slow down until he ate more carrots (carrots reduced his calorie intake). When we ran the simulation we saw that their maths was wrong and Santa would never go below 5,000 calories. He would never deliever all the presents!

So we decided he needed:

  • a warning light to flash at 4,750 calories
  • at this stage he should give the cookies to the reindeers until he burns more calories

But given that Santa needs to visit millions of homes in one night (24hours all in), he was still going too slow with the Chingu numbers. He must have a Hyper Drive and give something to the reindeers as a booster.

We added a ‘Fast Forward’ button to speed up the simulation. And whilst we were at it, a ‘Refresh’ button so that Santa and his elves could run simulations better.

As we’d given Santa a touchscreen dashboard we intended to use APIs to toggle the metrics with maps, his Spotify and a list of children, sorted by naughty and nice.

We started on the address book for presents for each child, but ran out of time.

Bugs and Solutions

There were quite a few along the way:

  • Tailwind is great, but I couldn’t add the background image. There is some conflict between CRA 4 and Tailwind 5. I tried running am alpha version of CRA 5, which is supposed to remedy this, but still didn’t have any success. It was quicker in the end to add the background image with old fashioned CSS
  • Our fast-forward button needed clicking twice to start it. But it only needed clicking once to turn it off, and from then on it only needed clicking once to start it. As I’m new to React I couldn’t fathom the issue; the code was exactly the same for all the other buttons.
  • Empty grid columns. I use these for padding along the sides, to keep all elements aligned. But I couldn’t control the minimum width with minmax(). They kept shrinking. I figured out I could add an empty character and set it’s colour to background colour. It’s a hacky solution; there must be a better way.
  • Santa’s list for naughty and nice children. We were fetching random names from an API, and assigning them presents. The idea was that 10 children would be added to the screen, at the top of a list, every second as the dashboard updated. But of course the API won’t allow so many frequent requests. So one giant fetch needs doing, then (but we ran out of time) probably mapping 10 names each second to a new array for the dashboard.
